AstroCRM · Build Plan · Lean Track

From a tool on your laptop
to a product clients log into

The app is already built as a single-user tool. The one thing missing is the layer that lets many clients sign in and each see only their own data. This is the plan to add it — leanly, for about €1 a month.

1Where things stand

More is built than it feels like. The gap is narrow and specific.

Already built & working

FastAPI app, 40+ endpoints: pipelines, deals, contacts (with import, profiles, file uploads), activity timeline, email templates, automations, campaigns via Resend, deadlines, export, lead capture. Plus the landing site, admin hub, and full business/architecture planning.

The missing piece

No login. No user accounts. No tenant separation — nothing in the code keeps one client's data apart from another's. The database lives as a single file on your laptop. This is the whole job of Phase 1.

Why this is the cheapest it will ever be: the app holds only 3 test contacts. There's no real client data to migrate or risk — so we build the multi-user foundation cleanly now, instead of retrofitting it painfully after 50 clients are on board.

3The build, in order

You = a quick, free account sign-up   Me = the coding. Nothing here touches real clients until Stage 6 passes.

  1. Create the accounts

    You · ~15 min · free

    Sign up for Supabase (database + login + backups) and Render (hosting), and connect Render to your GitHub. I'll walk you through each click.

  2. Rebuild the database with separation baked in

    Me

    Recreate your existing tables in Supabase, and add an account (workspace) to every one of them — so each row belongs to exactly one client. Switch on Row-Level Security so the database enforces it.

  3. Add login

    Me + Supabase

    A real sign-up and login screen. When someone registers, their private workspace is created automatically, with a starter pipeline. The app only opens after they log in.

  4. Make every feature account-aware

    Me

    Update all 40+ endpoints so each one only ever returns the logged-in client's own data. This is the largest coding step — it touches everything — but it's mechanical and low-risk.

  5. Put it online

    Me + your GitHub

    Move the app code to GitHub, connect Render so every change deploys itself, point it at Supabase, and set the address to app.astrocrm.app.

  6. Prove the walls hold

    Me

    Create two test clients, fill each with data, and confirm client A can never see client B — by any route. This is the go / no-go gate before anyone real touches it.

Then, before the first real client

One paid, one-off security review (~€300) — an outside expert confirms the isolation and login are watertight before real client data goes in. This is the single highest-value spend in the whole project. Not a salary, not a retainer — one check.
